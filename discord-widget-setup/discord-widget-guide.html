Discord Gamelist Widget, Step By Step

This is the plain-English version. Follow it in order and you will end up with a Discord profile widget that shows your current game and collection stats.

Game cover Currently Playing title Random game cover Three game rows Platform icons per row Trophy counts Collection slots Slot images

Before You Start

You need

A Discord account, access to the Discord Developer Portal, your Gamelist site URL, and a little patience. Discord widgets are still experimental.

You already have

Gamelist API routes for current games, completion stats, backlog, Shelf games, and trophy/achievement progress. The included script turns those into Discord widget data.

Original reference guide: Chloe Cinders' Discord widget guide.

Do not paste Discord bot tokens, Cloudflare secrets, or personal auth tokens into chat, GitHub, or public files.

Part 1: Make The Discord App

1
Open the Developer Portal

Go to discord.com/developers/applications.

Click: New Application
2
Name the app

This name appears on the widget, so use something nice like Shabii's Gamelist.

Click: Create
3
Enable Social SDK

Inside your app, look at the sidebar. Go to Games, then Social SDK. Fill the form Discord asks for.

If you skip this, the widget data scope may not work.

Part 2: Show The Hidden Widget Editor

1
Stay in the Discord Developer Portal

Open browser developer tools with F12 or Ctrl+Shift+I.

2
Open the Console tab

Use the experiment override code from Chloe's guide. Paste it only on the Discord Developer Portal page.

If Discord asks you to type allow pasting, type that first, then paste the code again.
3
Find the Widget menu

After the override works, go back into your app. You should see Games > Widget.

Click: Create Widget

Part 4: What The Script Reads From Gamelist

The script uses these URLs. You do not need to manually open them every time, but they are useful for testing.

YOUR_GAMELIST_BASE_URL/api/gamelist-games-by-list
Gets backlog count.
YOUR_GAMELIST_BASE_URL/api/completed-games-by-year
Gets total completed games and finished games this year.
YOUR_GAMELIST_BASE_URL/api/shelf-games-platforms
Gets Shelf game count.
YOUR_GAMELIST_BASE_URL/api/sync
Gets currently-playing games, covers, platforms, and started dates.
YOUR_GAMELIST_BASE_URL/api/achievements
Gets PlayStation activity summaries used as a progress fallback.
YOUR_GAMELIST_BASE_URL/api/trophies?id=GAME_ID
Gets PlayStation trophy counts when a selected game has a trophy title ID.
YOUR_GAMELIST_BASE_URL/api/steam-achievements?appId=APP_ID
Gets Steam achievement counts when a selected game has a Steam app ID.
YOUR_GAMELIST_BASE_URL/api/xbox-achievements?titleId=TITLE_ID
Gets Xbox achievement counts when a selected game has an Xbox title ID.
Use /api/... URLs. Do not use bare paths like /psn-trophies-by-year.

Part 5: Build The Widget Data

The single updater file, update-discord-widget.mjs, fetches your Gamelist info, writes Discord-ready JSON to widget-data.json, validates the published widget fields, and can send the update to Discord.

1
Open a terminal in the Gamelist folder

The folder is where this HTML file and update-discord-widget.mjs live.

2
Run this command 
$env:GAMELIST_BASE_URL = "https://your-gamelist-site.example"
$env:DISCORD_BOT_TOKEN = "YOUR_BOT_TOKEN"
$env:DISCORD_USER_ID = "YOUR_DISCORD_USER_ID"
node .\update-discord-widget.mjs --dry-run

The cover is randomized each time the script runs, and row 1 is the game shown on the cover.

3
You should now have a file

The file is called widget-data.json. This is what gets sent to Discord.

Part 6: Send The Data To Discord

Recommended path: use Chloe's Widget Identity Creator. It uses your bot token locally, fetches your application/widget config, validates the JSON against the published widget fields, and applies the identity data for you.

If you want to do the same thing manually, you need these values from Discord:

Application ID

Found in your app's Developer Portal overview.

Your Discord user ID

Enable Developer Mode in Discord, right-click your profile, copy user ID.

Bot token

Found under the Bot page. Keep this private. The desktop app keeps it on your machine.

widget-data.json

The file created in Part 5.

Authorize the application identity

If the PATCH request gives Invalid OAuth2 access token, open these URLs after replacing the application ID, then authorize while logged into the same Discord account as your user ID:

https://discord.com/oauth2/authorize?client_id=YOUR_APPLICATION_ID&response_type=token&scope=sdk.social_layer_presence&prompt=consent
https://discord.com/oauth2/authorize?client_id=YOUR_APPLICATION_ID&response_type=token&scope=openid%20sdk.social_layer_presence&prompt=consent
https://discord.com/oauth2/authorize?client_id=YOUR_APPLICATION_ID&response_type=token&scope=openid%20sdk.social_layer&prompt=consent

Chloe's app shows the first URL when Discord returns error 50025. Discord's Social SDK docs list openid with sdk.social_layer_presence as the default presence scopes, so the updater prints both versions. You usually also need the Social SDK form completed in the Developer Portal.

Paste this into PowerShell after replacing the placeholders:

$appId = "YOUR_APPLICATION_ID"
$userId = "YOUR_DISCORD_USER_ID"
$botToken = "YOUR_BOT_TOKEN"
$body = Get-Content .\widget-data.json -Raw

Invoke-RestMethod `
  -Uri "https://discord.com/api/v9/applications/$appId/users/$userId/identities/0/profile" `
  -Method PATCH `
  -Headers @{
    "Content-Type" = "application/json"
    "Authorization" = "Bot $botToken"
  } `
  -Body $body

Or use the local updater script

The updater builds fresh JSON, writes widget-data.json as UTF-8, checks your Discord widget config for field/type mismatches, and applies the identity.

$env:GAMELIST_BASE_URL = "https://your-gamelist-site.example"
$env:DISCORD_BOT_TOKEN = "YOUR_BOT_TOKEN"
$env:DISCORD_USER_ID = "YOUR_DISCORD_USER_ID"
node .\update-discord-widget.mjs

If Discord keeps returning 50025 after authorization, copy the access_token from the redirected OAuth URL and try the fallback:

$env:DISCORD_ACCESS_TOKEN = "ACCESS_TOKEN_FROM_REDIRECT_URL"
node .\update-discord-widget.mjs

To test without applying the update, add --dry-run.

If the command succeeds, do not paste the output publicly if it contains private data. Also never share the bot token.

Part 7: Keep It Updated

Manual refresh is enough at first. When you want to update your widget, run the updater again.

node update-discord-widget.mjs

Later, you can automate this once per hour with Task Scheduler, GitHub Actions, Cloudflare Workers, or another private scheduled job. If you have more than one currently-playing game, each automated update randomizes the rows and cover.

Do not update every minute. The Gamelist summary endpoints cache for one hour, and your Discord profile does not need instant updates.

Final Checklist

If Something Breaks

The widget says stats are still syncing

Make sure the PATCH request succeeded, your app was authorized correctly, and your widget Data Field names match the JSON names exactly.

One of the Gamelist URLs gives 404

Use /api/... paths and make sure the Cloudflare Worker has been deployed after route changes.

A subtitle only shows the game name

This is okay when that selected game does not have linked trophy or achievement data. For PlayStation, check the game has a trophy title ID. For Steam, check it has a Steam app ID. For Xbox, check it has an Xbox title ID.

The rows do not randomize

Randomizing happens when update-discord-widget.mjs runs. Make sure the automated job is running on a schedule and that more than one active game is marked as currently playing.
